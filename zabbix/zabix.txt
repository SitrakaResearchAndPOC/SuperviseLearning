Initialiser MySQL

# Lancer MySQL en arrière-plan (pas de systemctl/service dans Docker)
mysqld_safe &
sleep 10



mysql -u root -p <<EOF
CREATE DATABASE IF NOT EXISTS zabbix CHARACTER SET utf8mb4 COLLATE utf8mb4_bin;

USE zabbix;

CREATE USER IF NOT EXISTS 'zabbix'@'localhost' IDENTIFIED BY 'zabbixpass';
CREATE USER IF NOT EXISTS 'zabbix'@'127.0.0.1' IDENTIFIED BY 'zabbixpass';
GRANT ALL PRIVILEGES ON zabbix.* TO 'zabbix'@'localhost';
GRANT ALL PRIVILEGES ON zabbix.* TO 'zabbix'@'127.0.0.1';
FLUSH PRIVILEGES;

EOF

10️⃣ Créer un script pour démarrer tous les services
Créer /start-zabbix.sh :
cat << 'EOF' > /start-zabbix.sh
#!/bin/bash

# MySQL
mysqld_safe &

# Attendre MySQL
sleep 15

# Zabbix Server
/usr/sbin/zabbix_server -c /etc/zabbix/zabbix_server.conf &

# Zabbix Agent
/usr/sbin/zabbix_agentd -c /etc/zabbix/zabbix_agentd.conf &



Vérifier les logs

Pour suivre le démarrage :
docker exec -it zabbix-ubuntu tail -f /var/log/zabbix/zabbix_server.log


# Apache (frontend) au premier plan
apache2ctl -D FOREGROUND
EOF

chmod +x /start-zabbix.sh
./start-zabbix.sh






Apache tourne au premier plan, le conteneur reste actif.

MySQL, Zabbix Server et Agent tournent en arrière-plan.
